🏷️ 规则类型全览

DOMAIN-SUFFIX

DOMAIN-SUFFIX,google.com,Proxy

域名后缀匹配，最常用。匹配 google.com 及其所有子域名。

DOMAIN-KEYWORD

DOMAIN-KEYWORD,google,Proxy

域名包含关键词即命中，比后缀匹配更宽泛。

DOMAIN

DOMAIN,clashverge.org,Direct

精确域名匹配，不含子域名。

GEOIP

GEOIP,CN,Direct

基于 IP 地理数据库，判断流量归属地，通常用于国内直连。

IP-CIDR

IP-CIDR,10.0.0.0/8,Direct

匹配 IP 段，适用于局域网或特定 IP 范围。

MATCH

MATCH,Proxy

兜底规则，必须放在最后，匹配所有剩余流量。

SRC-IP-CIDR

SRC-IP-CIDR,192.168.1.0/24,Direct

基于客户端源 IP 进行分流，适合多设备策略。

PROCESS-NAME

PROCESS-NAME,firefox,Proxy

根据进程名匹配，指定特定软件走代理。

📌 匹配优先级

规则按 从上到下 的顺序匹配，一旦命中立即停止。因此务必遵循：

  • 最具体的规则放在最前面（如单个域名）
  • 广告屏蔽规则（REJECT）尽量提前
  • GEOIP 等宽泛规则居中
  • MATCH 作为最终兜底放在末尾

⚙️ 典型配置片段

# 广告拦截 DOMAIN-SUFFIX,doubleclick.net,REJECT # 国内域名直连 DOMAIN-SUFFIX,cn,DIRECT GEOIP,CN,DIRECT # 流媒体代理 DOMAIN-SUFFIX,netflix.com,🎬 流媒体 # 其余走代理 MATCH,🚀 自动选择

📦 进阶：Rule Provider 规则集

什么是 Rule Provider？

将大量规则托管在远程 URL 或本地文件，Clash 自动加载更新。无需在配置中堆积成千上万条规则，保持主配置清爽。

rule-providers: cn_domains: type: http url: "https://example.com/cn.yaml" path: ./rules/cn.yaml interval: 86400

如何使用规则集

在规则列表中使用 RULE-SET 引用：

rules: - RULE-SET,cn_domains,DIRECT - RULE-SET,ads,REJECT - MATCH,🚀 自动选择
💡 推荐使用社区维护的规则集（如 Loyalsoldier/clash-rules），每日更新，省心可靠。

❓ 规则配置常见问题

为什么规则不生效？

检查规则顺序，确保更精确的规则在上面。修改配置文件后需点击“重载配置”使其生效。

GEOIP 数据不准确？

Clash 使用的 GeoIP 数据库可能滞后。可以手动替换 Country.mmdb 文件或使用规则集代替。

如何让某个应用不走代理？

使用 PROCESS-NAME,yourapp.exe,DIRECT (Windows) 或 PROCESS-NAME,yourapp,DIRECT (macOS/Linux)。

MATCH 规则可以放在中间吗？

不可以。MATCH 会匹配所有流量，一旦出现在规则列表中间，后面的规则将永远不会被使用。
