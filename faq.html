代理速度慢

Windows 安装时提示“已阻止此应用”

因为应用未使用微软商店签名。点击“更多信息”，然后选择“仍要运行”。也可右键安装文件 → 属性 → 解除锁定。

macOS 提示“无法验证开发者”

前往“系统设置” → “隐私与安全性”，在底部找到“已阻止使用…”，点击“仍要打开”。或右键点击应用图标选择“打开”。

Linux 下 AppImage 无法运行

确保已安装 FUSE 库：sudo apt install libfuse2，并赋予执行权限 chmod +x ./Clash.Verge*.AppImage。

启动后没有窗口或闪退

尝试从终端启动查看错误日志。Windows 可查看事件查看器，macOS/Linux 在终端输入应用名运行。检查是否有其他代理软件冲突。

导入订阅后节点列表为空

检查订阅链接是否有效（浏览器打开看是否返回 YAML 内容）。部分机场需要订阅转换，或联系提供商获取 Clash 专用链接。

配置文件语法错误导致启动失败

使用在线 YAML 校验工具检查缩进和格式。常见错误：多余空格、缩进不一致、字段名拼写错误。

如何备份和恢复配置？

在配置页面右键选择“导出”，保存为 .yaml 文件。恢复时使用“从文件导入”。也可直接复制 ~/.config/clash-verge-rev 目录。

代理速度很慢或不稳定

尝试切换延迟更低的节点；检查策略组是否为 url-test 自动选择；关闭其他占用带宽的应用；确认本地网络正常。

某些网站无法访问（部分阻断）

可能是规则未覆盖。手动添加域名规则，或将该域名加入代理策略组。同时检查 DNS 是否返回了错误 IP。

手机/其他设备如何共享代理？

在 Clash Verge Rev 设置中开启“允许局域网连接”，其他设备将代理 IP 设为本机 IP，端口为 7890。

开启 TUN 模式后完全无法上网

首先确认已授予管理员权限。尝试在设置中切换 TUN 堆栈为 gvisor；检查 DNS 设置；重启客户端或电脑。

游戏或语音软件延迟高

确保 TUN 模式下 UDP 转发正常。可尝试为游戏单独添加直连规则，避免代理绕路。

虚拟网卡创建失败

Linux 需加载 tun 模块：sudo modprobe tun。Windows 检查是否有其他 VPN 软件占用，卸载 Wintun 驱动后重启。

如何检测 DNS 是否泄漏？

访问 ipleak.net 或 dnsleaktest.com，查看显示的 DNS 服务器。应只出现你配置的加密 DNS。

Fake‑IP 模式下某些局域网设备无法访问

将局域网域名（如 *.lan、*.local）加入 fake-ip-filter 列表，或在规则中对局域网 IP 段设置 DIRECT。

DNS 查询非常慢

检查配置的 DoH/DoT 服务器是否可达。可更换国内 DNS 服务商（如 doh.pub）作为备用。适当增大缓存时间。

🩺 快速诊断：我的代理为什么不工作？

1

检查系统代理是否开启

托盘图标右键 → 确保“系统代理”已勾选。浏览器才会使用代理。

2

验证节点是否可用

在“代理”页面点击节点测速，确认至少有一个节点延迟正常。

3

查看日志输出

打开“日志”页面，观察是否有红色错误信息，如“dial timeout”表示连接超时。

4

排除其他代理软件干扰

关闭其他 VPN/代理软件，避免端口冲突。重启电脑后仅启动 Clash Verge Rev。
