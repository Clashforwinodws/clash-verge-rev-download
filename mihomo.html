📜 起源与演进历程

2022 年 · Clash Meta 诞生

原版 Clash 内核逐渐停止维护，社区 fork 出 Clash Meta 分支，开始修复 bug 并增加新协议支持。

2023 年 · 协议大爆发

陆续加入 VLESS、Hysteria、TUIC 等新一代协议，成为协议支持最全面的代理内核之一。

2024 年 · 更名 Mihomo

为与“Meta”品牌区分，项目更名为 Mihomo，同时核心代码逐步向 Rust 迁移，性能再次飞跃。

2025‑2026 年 · 持续进化

引入更智能的规则引擎、QUIC 全链路支持、以及更完善的 Fake‑IP 实现，成为 Clash Verge Rev 的指定内核。

🚀 性能优势

  • ✅ Rust 语言编写，内存安全且零成本抽象
  • ✅ 异步 I/O 模型，轻松应对万级并发连接
  • ✅ 极低的 CPU 与内存占用，适合边缘设备

❓ 关于 Mihomo 的常见问题

Mihomo 和 Clash Meta 是什么关系？

它们是同一个项目。2024 年社区决定将 Clash Meta 更名为 Mihomo，以区别于 Meta 公司，代码仓库和功能完全一致。

如何确认我正在使用 Mihomo 内核？

在 Clash Verge Rev 中点击“关于”或查看日志，如果看到 Mihomo 或 Clash Meta 字样，说明已是该内核。

原版 Clash 的配置能在 Mihomo 上用吗？

绝大部分兼容。Mihomo 向下兼容 Clash 配置格式，同时扩展了部分字段。直接导入原配置通常可以正常工作。

如何参与 Mihomo 开发？

Mihomo 是开源项目，你可以在 GitHub 上提交 Issue 或 Pull Request。文档和贡献指南请查看 Wiki。
