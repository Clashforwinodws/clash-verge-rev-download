📥 导入订阅与配置

方法一：URL 导入（推荐）

大多数机场会提供一个 Clash 订阅链接。在 Clash Verge Rev 主界面点击左侧 “配置”，然后点击右上角 “新建” → “从 URL 导入”，粘贴你的订阅链接并保存。客户端会自动下载并解析节点信息。

💡 提示： 如果订阅链接无法直接访问，可尝试在前面添加 https://api.example.com/sub?target=clash&url= 作为代理转换。

方法二：手动粘贴配置

如果你已有完整的 YAML 配置文件，可在“配置”页面选择 “新建” → “手动编辑”，将内容粘贴到编辑器中并保存。配置文件通常包含 proxies、proxy-groups、rules 等字段。

方法三：本地文件导入

点击 “从文件导入”，选择一个本地 .yaml 或 .yml 文件。适用于已下载到本地的配置文件。

📝 配置文件结构解析

一个标准的 Clash 配置包含以下核心部分。理解这些字段是自定义规则的基础。

# 代理节点列表 proxies: - { name: "节点1", type: ss, server: ... } # 代理组（用于策略选择） proxy-groups: - name: "自动选择" type: url-test proxies: [节点1, 节点2] url: http://www.gstatic.com/generate_204 interval: 300 # 分流规则 rules: - DOMAIN-SUFFIX,google.com,🚀 自动选择 - GEOIP,CN,DIRECT - MATCH,🐟 手动选择

更多详细语法请参考 Wiki 文档。

📋 分流规则实战

常用规则类型

  • DOMAIN-SUFFIX：匹配域名后缀，如 DOMAIN-SUFFIX,youtube.com,Proxy
  • DOMAIN-KEYWORD：域名包含关键词
  • GEOIP：基于 IP 地理信息，如 GEOIP,CN,DIRECT
  • IP-CIDR：匹配 IP 段
  • MATCH：兜底规则，一般放在最后

典型配置示例

# 广告拦截 DOMAIN-SUFFIX,ad.com,REJECT # 国内直连 GEOIP,CN,DIRECT # 流媒体代理 DOMAIN-SUFFIX,netflix.com,🎬 流媒体 # 最终兜底 MATCH,🚀 自动选择

🌐 DNS 优化与防污染

启用加密 DNS

在配置文件中添加 dns 字段，使用 DoH 或 DoT 提升解析安全。

dns: enable: true listen: 0.0.0.0:53 enhanced-mode: fake-ip # 或 redir-host nameserver: - https://doh.pub/dns-query - tls://8.8.8.8 fallback: - https://cloudflare-dns.com/dns-query
🔍 Fake-IP 模式：将 DNS 解析结果映射为虚拟 IP，加速响应并防止 DNS 泄漏。适合大多数场景。

🔌 TUN 模式配置

全局流量接管

TUN 模式会在系统层创建虚拟网卡，接管所有应用的网络流量，无需为每个软件单独设置代理。在 Clash Verge Rev 中，只需在 “设置” → “TUN 模式” 中点击开启，并授予管理员/root 权限。

如需手动配置，可在配置文件中添加：

tun: enable: true stack: system # 或 gvisor dns-hijack: - any:53

开启后，请确保系统防火墙允许 Clash 通信。

🧠 智能策略组与自动切换

策略组类型

  • select：手动选择节点
  • url-test：自动选择延迟最低的节点（每300秒检测）
  • fallback：按顺序尝试，首个可用节点生效
  • load-balance：负载均衡

合理使用策略组可以让网络既快又稳。例如创建一个“流媒体”策略组，仅包含支持解锁的节点，并设置为 url-test。

❓ 配置常见问题

导入订阅后无节点显示？

检查订阅链接是否有效，尝试在浏览器中打开该链接，查看是否返回 YAML 内容。某些机场需要后端转换。

开启 TUN 模式后无法上网？

请确认已给予管理员权限，并检查 DNS 设置是否冲突。可尝试切换 TUN 堆栈为 gvisor。

如何让指定网站走直连？

在规则中添加 DOMAIN-SUFFIX,example.com,DIRECT，然后点击“重载配置”即可。

配置文件优先级？

Clash 会从上到下匹配规则，第一条匹配成功即停止。因此建议将精确规则放在前面，宽泛规则（如 MATCH）放在最后。
