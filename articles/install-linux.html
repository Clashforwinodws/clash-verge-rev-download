🐧 为什么在 Linux 上选择 Clash Verge Rev？

Linux 拥有丰富的代理工具生态，但 Clash Verge Rev 脱颖而出：它不仅提供了跨平台一致的现代图形界面，还内置了 Tauri 框架带来的轻量体验。与纯命令行的 v2ray、shadowsocks-libev 相比，Clash Verge Rev 的可视化配置、规则管理以及实时日志面板显著降低了使用门槛。特别是其集成的 Mihomo 内核 对 Linux TUN 模式的支持非常成熟，使得全局透明代理不再是 iptables 专家的专利。

无论你是使用 Ubuntu 进行日常开发，还是在 Fedora 上构建容器化环境，甚至是 Arch Linux 的滚动更新爱好者，本指南都将覆盖适合你的安装路径。后续的配置部分请参考 配置教程 以及 导入订阅指南。

📦 安装前准备：环境与依赖

Clash Verge Rev 的 AppImage 包几乎自包含所有库，但为了图形界面和 TUN 模式正常工作，仍需确保基础系统库已安装。不同发行版使用对应的包管理器：

# Ubuntu / Debian sudo apt update && sudo apt install -y libfuse2 libnss3 libgtk-3-0 libwebkit2gtk-4.1-0 # Fedora sudo dnf install -y fuse-libs nss gtk3 webkit2gtk4.1 # Arch Linux sudo pacman -S fuse2 nss gtk3 webkit2gtk-4.1

libfuse2 是运行 AppImage 的必要组件；webkit2gtk 则用于渲染客户端界面。如果你计划在无图形界面的服务器上运行，可以跳过 webkit2gtk，但后续配置可能需要通过另一台机器的浏览器访问（如使用 Clash Verge Rev 的远程控制功能，但目前版本仍以 GUI 为主）。此外，务必加载 tun 内核模块以备后续 TUN 模式：

sudo modprobe tun echo "tun" | sudo tee /etc/modules-load.d/tun.conf
💡 提示： 如果你使用的是 WSL2（Windows Subsystem for Linux），需要确保 WSL 内核支持 tun，并在 Windows 侧开启虚拟网卡功能。具体请参考 Ubuntu 部署教程 中的环境说明。

🗂️ 三种安装方式速览

根据你的使用习惯和发行版，可选择：

  • AppImage：通用格式，无需安装，下载即用，适合所有发行版。
  • 原生包（.deb / .rpm）：深度集成系统菜单，更新方便。
  • 用户仓库（AUR / COPR）：由社区维护，一行命令安装。

我们将在下面详细展开每种方法。

🎯 AppImage 通用安装

从 下载页面 或 GitHub Releases 获取最新的 Clash.Verge_*.AppImage 文件。赋予执行权限后即可运行：

chmod +x Clash.Verge_2.5.2_amd64.AppImage ./Clash.Verge_2.5.2_amd64.AppImage

首次启动可能需要几秒钟解压。如果系统提示缺少 FUSE，请回顾上文安装 libfuse2。若仍无法运行，可以尝试 ./Clash.Verge_2.5.2_amd64.AppImage --appimage-extract-and-run，这将直接解压并运行，绕过 FUSE 依赖。

为了日常使用方便，建议将 AppImage 移动到 /opt 并创建桌面快捷方式：

sudo mv Clash.Verge_2.5.2_amd64.AppImage /opt/clash-verge-rev.AppImage # 创建 .desktop 文件 cat > ~/.local/share/applications/clash-verge-rev.desktop << EOF [Desktop Entry] Name=Clash Verge Rev Exec=/opt/clash-verge-rev.AppImage Icon=clash-verge Type=Application Categories=Network; EOF

📀 发行版原生包安装

Ubuntu / Debian (.deb)

GitHub Releases 提供预编译的 .deb 包。下载后通过命令行或图形工具安装：

sudo dpkg -i ./Clash.Verge_2.5.2_amd64.deb # 如果提示依赖问题 sudo apt --fix-broken install

此后，Clash Verge Rev 将出现在系统菜单中，并可通过 apt 管理更新（需手动下载新版本）。

Fedora / RHEL (.rpm)

sudo rpm -i ./Clash.Verge_2.5.2_amd64.rpm

Fedora 用户也可使用 COPR 社区仓库（如 sudo dnf copr enable somebody/clash-verge-rev），以获得自动更新。

Arch Linux (AUR)

Arch 用户可以直接从 AUR 安装：

yay -S clash-verge-rev-bin

这将自动拉取最新的预编译包并处理依赖。

⚠️ 注意： 原生包通常不包含最新的 Mihomo 内核，你可能需要手动从 GitHub 下载内核文件并放入 ~/.config/clash-verge-rev/ 下，以确保协议支持完整。

🔧 TUN 模式与权限配置

Linux 下开启 TUN 模式需要管理员权限，因为要创建虚拟网络设备。最简单的方式是以 sudo 运行 Clash Verge Rev，但这会让整个进程拥有 root 权限，不够安全。推荐做法是赋予 AppImage 或可执行文件 CAP_NET_ADMIN 能力：

sudo setcap cap_net_admin+ep /opt/clash-verge-rev.AppImage

此后，普通用户启动时即可启用 TUN 模式。如果使用 .deb 安装，二进制文件通常位于 /usr/bin/clash-verge-rev，同样使用 setcap 指向它。

在 TUN 设置页面中，建议将堆栈设为 gVisor 以获得更好的兼容性，尤其对于自定义内核或受限的 VPS 环境。更多 TUN 模式的原理与调优，请阅读 开启 TUN 模式 和 TUN 断网排查指南。

🚀 使用 systemd 实现开机自启

在 Linux 上，通过创建用户级 systemd 服务可以让 Clash Verge Rev 随桌面登录自动启动，且无需 root 权限（除非需要 TUN，此时服务文件应使用系统级服务）。

创建 ~/.config/systemd/user/clash-verge-rev.service：

[Unit] Description=Clash Verge Rev 代理客户端 After=graphical-session.target [Service] Type=simple ExecStart=/opt/clash-verge-rev.AppImage Restart=on-failure Environment=DISPLAY=:0 [Install] WantedBy=default.target

然后启用服务：systemctl --user enable --now clash-verge-rev.service。如果 TUN 模式需要系统级权限，应将服务文件放在 /etc/systemd/system/，并在 [Service] 中添加 User=yourname，然后使用 sudo systemctl enable clash-verge-rev。详细步骤可参考 Ubuntu 部署教程 中的服务化章节。

🩺 常见问题与修复

1

AppImage 闪退

终端运行查看错误：可能是 libwebkit2gtk 缺失或版本不匹配。

2

TUN 设备创建失败

确认 tun 内核模块已加载，且已赋予 CAP_NET_ADMIN。

3

无法打开系统代理端口

检查 7890 等端口是否被占用：sudo lsof -i :7890，关闭冲突进程。

4

图形界面无法显示

SSH 远程需设置 DISPLAY 变量或使用 X11 转发。纯服务器建议配合无头内核。

如果你是从旧版本升级或从其他代理客户端迁移过来，请确保清理旧的系统代理设置，避免端口冲突。更多问题可查阅 常见问题 页面。
