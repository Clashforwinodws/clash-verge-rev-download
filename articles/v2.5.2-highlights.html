🚄 Hysteria2 协议：为高延迟网络而生

Hysteria2 是原 Hysteria 协议的全新迭代，基于修改版的 QUIC 协议（IETF 标准），专门为不稳定、高延迟的网络环境设计。与依赖 TCP 的传统协议不同，Hysteria2 在 UDP 上运行，并内置了智能拥塞控制算法。这使得它在跨洲际连接、移动网络或卫星链路上，表现远超 Shadowsocks 和 VMess。

核心优势

  • 抗丢包能力：即使丢包率达到 20%，连接依然稳定，不会出现 TCP 的“重传风暴”。
  • 低延迟：0-RTT 握手，连接建立速度比 TLS 快数倍。
  • 多路复用：一条连接承载所有流量，减少握手开销。
  • 端口跳跃：支持动态端口切换，绕过 QoS 限制。
💡 适合场景：跨国游戏加速、视频会议、实时语音传输。如果你在使用 TUN 模式 玩外服游戏时经常遇到卡顿，升级到 Hysteria2 节点会有明显改善。

在 Clash Verge Rev 中使用 Hysteria2 非常简单。只需确保你的机场或自建节点支持 Hysteria2 协议，然后将节点信息填入配置文件即可。以下是一个典型的 Hysteria2 节点配置片段：

# Hysteria2 节点示例 - name: "Hysteria2 节点" type: hysteria2 server: your-server.com port: 443 password: your-password up: 50 # 上传带宽 (Mbps) down: 200 # 下载带宽 (Mbps) sni: www.bing.com # 伪装 SNI

配置完成后，在 配置教程 中提到的策略组里选择该节点即可。Mihomo 内核会自动采用 QUIC 进行传输，你可以通过新连接面板直观地看到握手和传输过程。

📊 全新连接面板：让网络透明化

以往排查代理问题时，我们往往需要依赖日志文件，甚至是抓包工具。v2.5.2 引入的“连接状态”面板彻底改变了这一体验。在 Clash Verge Rev 左侧菜单点击 “连接”，你将看到一个实时更新的列表，展示所有活跃的代理连接。

面板信息包含：

  • 目标地址：流量最终要访问的域名或 IP。
  • 代理链：从本地应用到具体节点的完整路径。
  • 实时速度：当前连接的上传/下载速率。
  • 延迟 (RTT)：从发起请求到收到响应的耗时。
  • 规则命中：显示了该连接被哪条规则匹配（例如 “DOMAIN-SUFFIX,google.com,Proxy”）。

这项功能对调试 Rule 模式 的分流规则尤其有用。以前你可能困惑“为什么这个网站走了直连？”，现在只需打开面板，找到对应连接，就能看到究竟是哪条规则导致了该结果。此外，面板还支持按状态过滤（如只显示“失败”连接），快速定位问题。日常使用中，你也可以通过它监控哪些应用在大量消耗代理带宽，方便进行流量管理。

⚙️ 更多性能与稳定性改进

除了两大新功能，v2.5.2 还包含多项底层优化：

  • UDP 转发效率提升 15%：受益于 Mihomo 内核的异步 I/O 调整，游戏和语音应用的延迟明显降低，结合 TUN 模式 效果更佳。
  • Fake‑IP 内存占用降低：大规模规则集场景下，内存占用减少约 30%，低配设备也能流畅运行。有关原理可阅读 Fake‑IP 深度解析。
  • 订阅更新机制优化：修复了某些机场订阅解析失败的问题，并支持更灵活的更新间隔设置。
  • 配置快照功能：在配置页面可以创建当前配置的快照，若新配置出错，一键恢复。这为频繁修改规则的用户提供了安全网。
⚠️ 注意：如果你是从 v2.5.0 之前的版本升级，建议先备份配置文件。新版本对 dns 字段的部分默认值有微调，具体可查阅 更新日志。

🧪 配置示例与快速迁移

如果你已经拥有 Hysteria2 节点，可以直接在现有配置中添加 proxies 列表，然后在 proxy-groups 中引用。以下是一个结合了 Hysteria2 和自动选择策略的完整配置片段：

# 代理节点 proxies: - name: "Hysteria2 高速节点" type: hysteria2 server: proxy.example.com port: 443 password: your_pass up: 80 down: 300 # 策略组（自动选择最低延迟节点） proxy-groups: - name: "自动选择" type: url-test proxies: [Hysteria2 高速节点, 其他节点] url: 'http://www.gstatic.com/generate_204' interval: 300

保存后，打开连接面板观察新节点的延迟和速度。如果喜欢手动控制，可以将策略组类型改为 select，在界面上随时切换。更多的策略组玩法请参考 配置教程。

❓ 关于 v2.5.2 的常见问题

Q: Hysteria2 和 Hysteria 有什么区别？
Hysteria2 是全新的协议版本，不再兼容原版 Hysteria。它放弃了自定义的拥塞控制，转而采用标准 QUIC，在可靠性和穿透性上更优，且配置更简单。

Q: 连接面板会占用大量资源吗？
面板采用增量渲染，对 CPU 和内存的影响微乎其微。即使在低配设备上，同时显示数百条连接也不会感到卡顿。

Q: 升级后旧的节点还能用吗？
完全可以。v2.5.2 向下兼容所有旧协议节点，你只需在需要时添加 Hysteria2 节点即可。
